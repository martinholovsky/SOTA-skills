How it works · seamless skill invocation

Plain prompt in — the right skills load themselves

1 · YOU PROMPT

Plain language

No skill names, no trigger words, no stack required — your profile (or the skills' own selection rules) fills in the rest.

> Add file upload to our invoices API
→
2 · AUTO-ROUTING

Task → skills

The prompt is matched against every skill's description; the sota router maps task + mode (BUILD or AUDIT) to the domains it touches.

upload → security · API · language · tests
→
3 · SELECTIVE LOAD

Only what's needed

Each skill's index picks the matching rules files (every file < 500 lines) — never the whole library, so context stays lean.

SKILL.md → rules/09-ingestion … nothing else
→
4 · APPLIED

Build · audit · verify

BUILD applies the rules while writing and self-checks the diff against each audit checklist; AUDIT emits file:line | rule | severity | effort | fix.

✓ hardened upload + tests + checklist pass
Loaded for this prompt code-security api-design python testing 4 of 40+ skills — routing is automatic; an optional always-on directive makes it phrasing-independent
